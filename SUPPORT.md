# Support Guide

Thank you for using AxCom! This document directs you to the right channels for assistance.

---

## 📖 1. Documentation

Before seeking support, please search the official documentation. The answers to many questions can be found there:

- **Documentation Site:** [docs.AxCom.dev](https://docs.AxCom.dev)
- **Getting Started Guide:** [Getting Started](docs/getting-started/intro.md)
- **API Reference:** [API Reference Documentation](docs/api/)

---

## 💬 2. Community Q&A & Discussions

For general questions, architectural advice, integrations, or sharing ideas:

- **GitHub Discussions:** [AxCom Discussions](https://github.com/axiolon/axcom/discussions)
  - Use the **Q&A** category to ask questions.
  - Use the **Ideas** category to suggest enhancements before opening formal feature requests.

---

## 🐛 3. Reporting Issues & Requesting Features

If you have confirmed a bug or want to request a new feature:

- **GitHub Issues:** [AxCom Issues](https://github.com/axiolon/axcom/issues)
- Please use the appropriate issue form (`Bug Report` or `Feature Request`) and fill out all required details.

---

## 🔒 4. Security Vulnerabilities

If you believe you have found a security vulnerability:

- Please **do not** open a public issue.
- Refer to our [Security Policy](SECURITY.md) for reporting instructions.
- Email details to: **security@axiolon.com**

---

## 💼 5. Commercial Support

For custom development, enterprise deployment services, and SLAs:

- **Contact Website:** [axiolonlabs.dev](https://axiolonlabs.dev)
- **Support Email:** support@axiolon.com
